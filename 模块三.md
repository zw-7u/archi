# 模块三《巧物精工》搭建 README

## 1. 模块定位

模块三《巧物精工》用于展示故宫建筑中“可被拆解、可被聚焦、可被解读”的重点构件逻辑。页面以“建筑切换 + 全景拆解 + 构件解读”作为主线，顶部负责切换建筑，中部负责构件热区交互与拆解展示，右侧负责当前建筑的重点构件按钮、构件图片轮播与三维度知识解读。

本模块以你最新的需求为准：
- 顶部为四个建筑切换按钮：角楼、午门、太和殿、九龙壁。
- 中间为全景大屏：底层使用 `png`，上层叠加同名 `svg` 路径热区作为可交互拆解层。
- 点击热区后，对应构件从建筑主体中“拉出”到旁侧，并显示构件名称。
- 全景区右上角提供“一键拆分”按钮，触发整体拆解状态：`png` 缩到左上角，`svg` 各构件按预设位置散开，并统一显示名称。
- 全景区底部保留两个按钮：`3D全景`、`巧筑工坊`，当前只搭建 UI 与占位逻辑，后续再接事件。
- 右侧为“重点构件解读”区，包含三个层级：构件按钮组、构件图片轮播/展示区、三标签知识卡片。

以上结构来自你上传的模块三说明与构件卡片内容文件。fileciteturn4file0 fileciteturn4file1

## 2. 页面布局总览

整体页面采用 **上方建筑切换 + 中部主工作区 + 右侧解读栏** 的三段式结构。

### 2.1 页面分区

#### A. 顶部建筑切换栏
位于主内容区顶部横向排列，放置四个建筑按钮：
- 角楼
- 午门
- 太和殿
- 九龙壁

每个按钮由两部分组成：
- 对应建筑的 SVG 图标
- 建筑名称文字

点击按钮后需要完成以下同步切换：
1. 中部全景 `png` 切换为当前建筑图
2. 中部交互 `svg` 切换为当前建筑路径图
3. 右侧重点构件按钮切换为当前建筑对应构件列表
4. 右侧图片轮播切换为当前建筑对应构件图集合
5. 右侧底部知识卡默认切到当前建筑的第一个构件
6. 如果当前处于“一键拆分”状态，切换建筑时先重置到默认聚合状态

#### B. 中部全景交互区
位于页面主体中央，占据主要视觉焦点。

包含以下元素：
- 建筑全景底图 `png`
- 覆盖在上方的构件热区 `svg`
- 右上角浮动按钮：`一键拆分`
- 底部两个功能按钮：`3D全景`、`巧筑工坊`
- 构件名称标签层
- 构件拆解动画层

#### C. 右侧重点构件解读区
位于右侧固定信息栏，分为三块：

1. **构件按钮区**
   - 根据当前建筑动态生成按钮
   - 一个建筑有几个构件，就生成几个按钮

2. **构件图片展示区**
   - 默认自动轮播当前建筑相关构件图片
   - 点击某个构件按钮后，立即切换展示该构件对应图片
   - 图片路径使用 `images/pattern/构件名.jpg`

3. **下方知识卡片区**
   - 标题：`重点构件解读`
   - 内部是三个标签页：
     - 形制与工艺
     - 物理与数学
     - 文化与符号
   - 卡片内容来源于 `kapian.md`

### 4.2 推荐转成结构化 JSON

不建议每次运行时直接在页面里硬解析原始 Markdown。更稳妥的方式是：
1. 先手动整理一次
2. 转成单独的 `module3-data.js`

```js

      {
        id: "jishou",
        name: "脊兽",
        image: "images/pattern/脊兽.jpg",

        tabs: {
          craft: "...",
          physics: "...",
          culture: "..."
        }
      }
    ]
  }
}
```

其中 `explode` 字段用于控制：
- 单击构件时拉出的位置
- 一键拆分时散开的位置

这样可以保证两种状态位置一致，符合你“散开位置和鼠标点击拉出一样”的要求。fileciteturn4file0

## 7. 交互逻辑设计

## 7.1 顶部建筑切换逻辑

### 默认状态
- 默认进入模块三时显示你指定的首个建筑，建议为 **太和殿**，因为它构件最丰富，展示效果最好。
- 若你希望延续你此前页面逻辑，也可以默认显示“角楼”，但从信息密度和演示角度，太和殿更适合作为初始页。

### 点击建筑按钮后
触发以下流程：

1. 更新 `currentBuilding`
2. 重置 `selectedComponent = 当前建筑第一个构件`
3. 重置 `isExploded = false`
4. 替换 `png`
5. 重新加载对应 `svg`
6. 重新生成右侧构件按钮
7. 重新启动当前建筑构件图轮播
8. 右下卡片展示当前选中构件的三个维度内容

## 7.2 中部热区 hover / click 逻辑

### hover 状态
鼠标移入某个 `path` 时：
- 该 path 高亮
- 显示构件名称浮签
- 鼠标样式切换为可点击状态

### click 状态
点击某个 `path` 时：
- 该构件从原始建筑位置平滑拉出到预设散开位置
- 同时显示该构件名称标签
- 右侧构件按钮切换为当前构件激活态
- 右侧图片区切换为该构件图片
- 右下知识卡切换为该构件内容

### 再次点击同一构件
建议处理为：
- 若当前不是整体拆分状态，则再次点击收回该构件
- 若当前处于整体拆分状态，则单个构件不允许收回，只允许通过“还原”或切换建筑整体复位

这里建议把“一键拆分”按钮做成 **二态按钮**：
- 初始：一键拆分
- 拆分后：还原结构

虽然你目前只明确写了“一键拆分”，但从可用性和调试角度，加一个“还原”状态会更合理。

## 7.3 一键拆分逻辑

点击全景区右上角按钮后：

### 进入拆分状态
1. `png` 缩小并移动到左上角
2. 所有可拆解 `path` 按预设 `explode` 坐标散开
3. 每个构件旁显示名称标签
4. 当前选中构件同步更新为第一个构件，或保持用户点击前选中的构件
5. 右侧内容不消失，继续显示当前选中构件信息

### 退出拆分状态
1. 所有 path 回归原位置
2. 构件名称标签隐藏
3. `png` 回到默认居中放大状态
4. 保留当前构件选中信息

### 位置规则
一键拆分后的散开位置，必须与单击构件后拉出的落点保持一致。这一点直接写入数据层，不要在代码里分两套坐标。fileciteturn4file0

## 8. SVG 热区实现要求

这是模块三最关键的实现前提。

### 8.1 SVG 中每个可交互构件必须具备独立 path 或 group

例如：

```html
<g id="dougong" data-name="斗拱">
  <path d="..." />
</g>
```

或者：

```html
<path id="dougong" data-name="斗拱" d="..." />
```

### 8.2 路径命名必须与数据层映射一致

建议在数据层明确记录：

```js
{
  id: 'dougong',
  name: '斗拱',
  svgTarget: 'dougong'
}
```

不要依赖中文名直接去找 DOM，避免空格、特殊字符、导出后路径名变化带来问题。

### 8.3 允许一个构件由多个 path 组成

如果一个构件由多个碎片路径构成，则建议：
- 在 SVG 里用 `<g>` 包一层
- 事件绑定到 `g`
- 拆解时整体移动 `g`

这样更稳定，也更适合复杂构件。

## 9. 状态管理建议

建议在 `module3-state.js` 维护以下核心状态：

```js
export const module3State = {
  currentBuilding: '太和殿',
  selectedComponentId: null,
  activeInfoTab: 'craft',
  isExploded: false,
  autoPlayEnabled: true,
  loadedSvgMap: {},
};
```

字段说明：
- `currentBuilding`：当前建筑
- `selectedComponentId`：当前选中构件
- `activeInfoTab`：当前知识卡标签
- `isExploded`：是否整体拆分
- `autoPlayEnabled`：是否允许图片自动轮播
- `loadedSvgMap`：缓存已加载过的 SVG，避免重复请求

## 10. 视觉与动效建议

## 10.1 整体风格
延续你项目已有的典雅、素净、古风方向，但模块三应更突出“工艺解剖感”和“精工细部感”。

建议关键词：
- 温润米白 / 浅金 / 黛青 / 朱砂点缀
- 轻磨砂面板
- 细线描边
- 微金色高亮
- 轻阴影层次

## 10.2 动效建议

### 构件 hover
- 微发光
- 描边变亮
- 浮签淡入

### 构件 click 拉出
- 使用 `transform: translate(...) scale(...)`
- 缓动建议：`cubic-bezier(0.22, 1, 0.36, 1)`
- 时长：`500ms ~ 700ms`

### 一键拆分
- `png` 先缩小，再 path 散开
- 不建议所有元素完全同时动，可错开 `80ms ~ 120ms`
- 整体更有“拆解展示”感

### 右侧按钮切换
- 当前构件高亮底色
- 轻微外发光
- 不要做过强跳动动画

## 13. 关键实现注意事项

### 13.1 不要让路径名和构件名脱节
最容易出问题的地方是：
- SVG 的 path id 叫 `path12`
- 数据层写的是“斗拱”
- 最后无法联动

所以必须建立清晰映射：

```js
svgTarget: 'dougong'
name: '斗拱'
```

### 13.2 不要在拆解时真的删掉 path
建议只改 `transform` 或包裹层的位置，不要重新创建 DOM，否则容易导致 hover、click、状态同步混乱。

### 13.3 标签层单独放置
构件名称标签不要直接写进 SVG，建议单独做一个 `label-layer`，这样更方便控制显隐、动效和样式。

### 13.4 图片缺失要允许降级
如果 `images/pattern/构件名.jpg` 暂时不存在：
- 使用默认占位图
- 控制台提示缺图
- 页面不报错不白屏

### 13.5 构件少的建筑也要保持界面完整
像角楼目前只有一个构件，右侧按钮区不能显得太空。建议：
- 单个按钮也保持完整样式
- 图片区和卡片区高度比例不变
- 不因为数据少就破坏整体版式
