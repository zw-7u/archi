# 故宫建筑信息可视化设计 —— 技术实现与作品亮点

## 一、作品定位与概述

### 1.1 作品名称
**《宫阙载史 · 堪舆哲思 · 巧物精工》—— 故宫建筑数字人文交互图谱**

### 1.2 赛道背景
本作品参赛类别为"留学生 + 汉学"赛道，面向：
- 在华留学生群体，辅助其理解中国古建筑文化
- 海外汉学研究者，提供可深度探索的建筑知识图谱
- 普通公众，兼顾专业性与可读性

### 1.3 核心目标
将故宫建筑群从静态文物转化为**可交互、可探索、可学习的数字人文知识系统**，通过空间可视化、哲学解读与构件解构三个维度，让用户理解故宫不仅是建筑集合，更是中国礼制文化、宇宙观与工匠智慧的物化表达。

## 三、核心技术实现

### 3.1 模块一：宫阙载史

#### 3.1.1 SVG 总览图 + 热区交互

**技术方案：** 动态加载外部 SVG 文件，通过 `svgScanner.js` 扫描热区节点，自动注入交互层与叠加层。

**核心流程：**

```
fetch('overview.svg')
  → 解析 SVG DOM
  → 创建 <g data-layer="hotspot-interaction">  交互层
  → 创建 <g data-layer="hotspot-overlay">      叠加层
  → 为每个热区绑定 mouseenter / mouseleave / click
  → CSS 控制叠加层样式（高亮、脉冲、选中态）
```

**热区数据结构：**

```javascript
{
  hotspotId: 'taihedian',   // 热区唯一标识
  buildingId: 'taihedian',   // 关联建筑 ID
  labelZh: '太和殿',        // 中文标签
  labelEn: 'Hall of Supreme Harmony',
  overlayEl: SVGPathElement, // 叠加层元素
  sourceEl: SVGPathElement, // 交互源区域
}
```

**高亮逻辑：** 支持多层级同时高亮——功能筛选高亮、形制筛选高亮、建筑选中态可叠加渲染，通过 CSS 自定义属性 `--highlight-color` 实现颜色驱动。

#### 3.1.2 ECharts 力导向关系网络

**技术方案：** 使用 ECharts 5 的 `graph` 类型，`layout: 'force'` 力导向布局，展示建筑在宫城空间中的关系网络。

**节点设计：**

- 中心节点（金色）：当前选中建筑，`symbolSize: 64`，带发光阴影
- 关联节点（彩色）：通过 `NODE_META` 分类着色，`symbolSize: 38`
- 关系线：相关系数 `value > 3` 时线条加粗，`curveness: 0.2` 弧度

**动态更新：** 每次建筑切换时重建图数据，`animationDuration: 600ms` 平滑过渡，点击节点可跳转建筑档案。

#### 3.1.3 三视图档案卡系统

每座建筑提供三个视角的档案：

| 视图 | 内容重点 | 交互特征 |
|------|----------|----------|
| **建筑卡** | 建筑身份、建造时间线、材料结构、形制样式 | 静态数据展示 |
| **事件卡** | 历史事件、节庆礼制、逸闻掌故 | 时间线排列 |
| **汉学卡** | 留学生友好解读、留学生提示、关键词解析 | 含"给留学生的提示"特别内容 |

汉学卡内容针对留学生设计，使用类比手法（如"太和殿就像一个公司的总部会议室"）降低文化理解门槛，并提供中英双语。

### 3.2 模块二：堪舆哲思

#### 3.2.1 阴阳五行模块（yinyang.js）

**PNG + SVG Overlay 双层架构：**

```
thought-stage
  ├── img.yy-base-map         （PNG 底图，故宫实景）
  ├── svg.yy-overlay-svg      （SVG 叠加层，可交互）
  │     ├── zones-yinyang     （阴阳分区路径）
  │     ├── zones-wuxing      （五行分区路径）
  │     └── yy-relation-lines （关系箭头线）
  └── div#yy-anchor-labels    （方位标签 HTML 层）
```

**Canvas 阴阳表盘：** 使用 Canvas 2D API 绘制双表盘（阴阳 + 五行），高 DPI 适配（`devicePixelRatio`），支持动态颜色映射。

**分区高亮逻辑：** 点击右侧按钮时，对应分区路径的 `fill-opacity` 由 `0.001` 变为 `1`，并添加 `stroke` 描边；非活跃分区添加 `yy-dim` 类实现视觉压暗。

#### 3.2.2 风水格局模块（fengshui.js）

**SVG Overlay 显隐控制：**

每个主题（front / water / orientation / celestial / guard）定义三层 overlay 集合：

```javascript
{
  overlays: ['front-region', 'front-axis'],      // 完全显示
  overlaysDim: ['mountain-arc', 'water-line'],    // 压暗显示
  overlaysHide: ['celestial-dots', 'guard-zone'], // 完全隐藏
  hotspots: ['front-hotspot', 'gate-hotspot'],   // 可交互热区
}
```

**Canvas 八卦方位盘：** 绘制北/东/南/西/中五个方位节点，活跃节点带光晕效果；支持四种流线类型（southToCenter / curveToCenter / eastWestWrap / northToCenter）。

**Canvas 气脉势能雷达图：** 五维雷达图展示当前区域的气脉评分，使用 `requestAnimationFrame` 实现从当前值到目标值的平滑插值动画（`qiCurrent[i] += diff * 0.12`）。

#### 3.2.3 中轴礼序模块（module2.js）

**礼序刻度轴可视化：** CSS + HTML 实现从南到北的建筑序列节点动画，关键节点（午门 → 太和殿 → 乾清宫 → 神武门）以递进动画方式点亮。

### 3.3 模块三：巧物精工

#### 3.3.1 SVG 构件拆分动画

**技术方案：** 建筑 SVG 中的每个构件（脊兽、斗拱、琉璃瓦等）为独立 `<g>` 元素，点击"一键拆分"后通过 CSS `transform` 实现爆炸视图：

```javascript
// 方向向量计算
const vectorX = selfX - centerX
const vectorY = selfY - centerY
const distance = Math.hypot(vectorX, vectorY)
// 从中心向外的爆炸位移
translateX = (vectorX / distance) * baseX * 1.5
translateY = (vectorY / distance) * baseY * 1.5
transform: translate(tx, ty) scale(1.08)
// 贝塞尔缓动曲线
transition: transform 0.5s cubic-bezier(0.22, 1, 0.36, 1)
```

**动画曲线选择：** `cubic-bezier(0.22, 1, 0.36, 1)` 为缓出曲线，模拟物理惯性效果，构件从中心向外扩散时有轻微减速感。

#### 3.3.2 Three.js 3D 模型查看器

**技术方案：** 使用 Three.js 的 GLTF/GLB 模型加载器，配合 `OrbitControls` 实现轨道控制：

- 拖拽旋转（鼠标左键）
- 滚轮缩放（鼠标滚轮）
- 右键平移（鼠标右键）

模型加载进度由 `LoadingManager` 统一管理，加载中和错误状态均有 UI 反馈。

#### 3.3.3 三维构件知识卡片

每个构件提供三个维度的深度解读：

| 维度 | 关注点 | 示例（以斗拱为例）|
|------|--------|-------------------|
| **形制与工艺** | 外观、材料、工艺流程 | 块、拱、昂层层叠加，榫卯拼接 |
| **物理与数学** | 结构受力、几何逻辑、材料特性 | 斗拱如悬臂梁分散荷载，模块重复 |
| **文化与符号** | 等级制度、哲学隐喻、礼制规范 | 九踩斗拱标志皇家最高等级 |

### 3.4 AI 智能助手（逐令助手）

**技术方案：** 基于 RESTful API 的智能助手，支持中英双语对话：

- 后端 API 网关（`server/server.js`）
- 前端助手组件（`assistant-widget.js`）
- 语音合成模块（`assistant-speech.js`）
- 对话管理（`assistant-api.js`）

用户可通过悬浮小助手随时提问，获得关于故宫建筑的即时解答。

### 3.5 中英双语系统

**实现机制：** 全局状态 `state.lang` 控制语言切换，CSS `data-lang` 属性驱动样式变化，所有文本内容通过 `pick(zh, en)` 函数根据当前语言返回对应版本。

```javascript
function pick(zh, en) {
  return isZh() ? zh : en
}
```

所有 HTML 文本均支持双语，包括热区标签、档案内容、构件说明、模块标题等。

## 四、作品亮点

### 4.1 创新点

#### 亮点一：空间叙事 + 数据可视化的深度融合

不同于传统文物展示的"图片 + 文字"模式，本作品将故宫的空间组织（中轴、阴阳、五行、风水）转化为**可交互的空间叙事系统**：

- 阴阳分区、五行落位、风水格局不再停留在文本描述，而是变成可点击、可高亮、可动画的可视化图层
- 用户可以通过亲身"操作"来理解抽象哲学概念——点击"木"区域，对应方位亮起；切换到"相生"视图，关系箭头动画流转

#### 亮点二：留学生友好型汉学内容设计

针对留学生群体的特殊需求，设计了多层降维解释策略：

- **汉学卡**：每座建筑配有"给留学生的提示"，使用生活化类比（如"太和殿就像公司的总部会议室"）降低理解门槛
- **汉学解析段落**：堪舆哲思模块配有 Sinology Insights 区块，将中轴、阴阳、风水等概念用通俗语言重新诠释
- **中英双语完整内容**：所有档案内容均有高质量英文翻译，非机械翻译，而是考虑了文化背景的意译

#### 亮点三：构件级建筑知识解构

模块三"巧物精工"实现了中国古建筑首次从**构件视角**进行交互化解构：

- 将建筑拆解为脊兽、斗拱、琉璃瓦、台基等独立构件
- 每个构件提供形制、工艺、物理、文化四个维度的深度解读
- "一键拆分"动画让用户直观看到构件的空间关系与组装逻辑

#### 亮点四：多模态交互体验

系统整合了多种交互模态，形成丰富的学习体验：

- **热区点击**：SVG 地图直接交互，直观高效
- **Canvas 动画**：阴阳表盘、气脉雷达图、八卦方位盘提供动态反馈
- **3D 模型**：Three.js 全景查看器，支持旋转、缩放、平移
- **交互工坊**：构件拼装游戏，边玩边学
- **AI 对话**：智能助手提供即时问答支持

### 4.2 技术亮点

#### 亮点五：SVG + Canvas 混合渲染架构

在堪舆哲思模块中创新性地将 PNG 底图、SVG 交互层、HTML 标签层、Canvas 动画层四者叠加：

- PNG 提供高精度实景图像
- SVG 提供精确的交互热区和矢量高亮
- HTML 提供与用户界面风格统一的标签元素
- Canvas 提供高性能的实时动画效果

这种混合架构兼顾了视觉效果、交互精度和动画性能。

#### 亮点六：CSS 驱动的可视化状态系统

利用 CSS 自定义属性实现可视化状态的动态控制：

```css
.zone.yy-glow {
  fill: var(--yy-glow, currentColor);
  filter: drop-shadow(0 0 8px var(--yy-glow));
}
```

通过 JavaScript 动态修改 CSS 变量（如 `--yy-glow`、`--highlight-color`），实现无需重绘 DOM 的流畅高亮切换。

#### 亮点七：模块化子模块插件系统

堪舆哲思模块采用"模块注册表"模式，子模块（module2 / yinyang / fengshui）作为独立插件接入主系统：

```javascript
// 主系统提供标准接口
window.YinyangModule = { selectBtn, render, renderButtonsOnly, state }
window.FengshuiModule = { init, renderButtonsOnly, renderAll, setTheme }

// 主模块在渲染时检测并调用
if (state.thoughtTab === 'yinyang' && typeof YinyangModule !== 'undefined') {
  YinyangModule.render()
}
```

这种设计允许各子模块独立开发、测试，同时通过标准化接口与主系统无缝集成。

#### 亮点八：响应式 Canvas DPI 适配

所有 Canvas 图表均实现了高 DPI 屏幕适配：

```javascript
const dpr = window.devicePixelRatio || 1
canvas.width = w * dpr
canvas.height = h * dpr
ctx.scale(dpr, dpr)
canvas.style.width = w + 'px'
canvas.style.height = h + 'px'
```

确保在 Retina/HiDPI 屏幕上图表同样清晰锐利。

### 4.3 应用价值与特色

#### 价值一：数字人文研究的可视化工具

作品不仅是展示工具，更是一套可深度探索的研究级知识图谱：

- 建筑关系网络揭示了故宫的空间组织逻辑
- 堪舆哲思模块将抽象哲学概念转化为可操作的交互系统
- 构件数据库积累了完整的建筑构件知识体系

#### 价值二：跨文化传播的教育平台

面向留学生的设计理念贯穿始终：

- 汉学卡特别内容降低文化理解门槛
- 完整中英双语支持无语言障碍
- 多维度解读满足不同认知风格（视觉型/阅读型/动手型）

#### 价值三：中国古建筑数字化保护的示范

作品展示了中国古建筑数字化保护的创新方向：

- 从"图片存档"到"知识图谱"的升级
- 从"静态展示"到"交互探索"的转变
- 从"单一视角"到"多维解读"的拓展
