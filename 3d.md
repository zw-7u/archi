# 巧物精工模块 3D 模型接入 README

## 一、任务目标

在现有故宫数字化交互项目中，为 **“巧物精工”** 模块接入 4 个 `glb` 三维模型，并将它们绑定到各建筑下方的 **“3D全景”** 按钮。

实现效果如下：

1. 用户进入 **巧物精工** 模块后，页面原有布局保持不变。
2. 每个建筑（如角楼、午门、太和殿、九龙壁）下方已有一个 **“3D全景”** 按钮。
3. 点击该按钮后，页面中央弹出一个 **3D 模型查看器弹层**。
4. 弹层内加载该建筑对应的 `.glb` 模型。
5. 用户可进行最基础的交互：

   * 鼠标左键拖拽旋转模型
   * 鼠标滚轮缩放
   * 右键拖拽平移（如使用 OrbitControls 默认行为）
6. 弹层支持关闭。
7. 关闭弹层时销毁当前 three.js 实例与模型资源，避免重复打开后越来越卡。
8. 本次只做最基础版本，**不做构件热点、不做标注、不做拆分、不做自动讲解**。

---

## 二、项目背景与限制

这是一个已有页面结构和 UI 风格的项目，当前重点是 **在不破坏已有模块布局和功能的前提下**，新增 3D 模型查看功能。

### 必须遵守的限制

1. **不要改乱现有巧物精工页面的布局和逻辑**
2. **不要影响已有的建筑切换、右侧卡片、全景底图、SVG 热区、巧筑工坊按钮等功能**
3. 只为 **“3D全景”** 按钮增加点击事件和弹层查看器
4. 不要一次性预加载全部模型，必须采用 **点击后按需加载**
5. 不要把 3D 模型直接常驻嵌入页面中间区域，必须采用 **弹层模式**
6. 若某个模型加载失败，需要给出简单提示，不能让页面报错卡死
7. 尽量复用一套查看器逻辑，而不是为四个建筑各写一套重复代码

---

## 三、模型资源路径

模型文件位于项目根目录下的 `models/` 文件夹中，文件路径形式如下：

```txt
models/jiaolou.glb
models/wumen.glb
models/taihedian.glb
models/jiulongbi.glb
```

如果当前项目中的实际文件名略有不同，请以项目真实文件名为准，但统一通过一个配置表进行映射，不要把路径硬编码分散写在多个地方。

---

## 四、推荐技术方案

使用以下技术实现：

* `Three.js`
* `GLTFLoader`
* `OrbitControls`

采用 **一个统一的 3D 查看器弹层** 来承载所有模型。

即：

* 点击哪个建筑的 “3D全景” 按钮
* 就读取该按钮对应的 `data-model`
* 根据映射表加载对应的 `.glb`
* 在同一个弹层组件内展示

## 五、需要完成的功能

## 1. 为 3D 全景按钮绑定模型类型

请检查巧物精工模块中四个建筑下方的 **“3D全景”** 按钮。
如果当前按钮没有标识，请为每个按钮补充统一的 class 和 `data-model` 属性，例如：

```html
<button class="btn-model-view" data-model="jiaolou">3D全景</button>
<button class="btn-model-view" data-model="wumen">3D全景</button>
<button class="btn-model-view" data-model="taihedian">3D全景</button>
<button class="btn-model-view" data-model="jiulongbi">3D全景</button>
```

要求：

* 保留原按钮视觉样式
* 若已有 class，不要粗暴覆盖，尽量追加
* 不要影响“巧筑工坊”按钮

---

## 2. 新增统一的 3D 弹层查看器

在页面中新增一个默认隐藏的弹层结构，用于显示 3D 模型。

弹层应包含：

1. 半透明遮罩层
2. 居中的查看器面板
3. 标题区（显示当前建筑名称）
4. 关闭按钮
5. 模型渲染区域
6. 一个最基础的底部提示文案，例如：

   * “拖拽旋转，滚轮缩放”

弹层默认隐藏，点击按钮后显示，点击关闭后隐藏。

---

## 3. 使用 three.js 加载 glb 模型

当用户点击某个按钮后：

1. 根据 `data-model` 读取对应模型 key
2. 在配置表中找到：

   * 标题
   * 模型路径
   * 相机初始位置
   * controls target
   * 可选的模型缩放比例
3. 初始化 three.js 场景
4. 使用 `GLTFLoader` 加载对应 glb
5. 将模型加入场景
6. 启动渲染循环

必须支持最基础交互：

* 旋转
* 缩放
* 平移

---

## 4. 关闭时释放资源

关闭弹层时必须正确清理：

1. 停止动画帧
2. 销毁 controls
3. 销毁 renderer
4. 将场景中的当前模型移除
5. 对 geometry 和 material 做 dispose
6. 清空 canvas 容器
7. 防止反复打开后内存累积

这是本任务的重点之一，必须认真处理。


  * 打开/关闭弹层
  * 调用查看器实例

如果当前项目没有 `modules/` 目录，也可以在合适位置新增，但要保持结构清晰。

## 九、three.js 具体实现要求

请基于 `Three.js + GLTFLoader + OrbitControls` 实现。

### 需要包含的基础内容

1. `Scene`
2. `PerspectiveCamera`
3. `WebGLRenderer`
4. `AmbientLight`
5. `DirectionalLight`
6. `OrbitControls`
7. `GLTFLoader`

### 基础场景要求

* 背景可用深色
* 模型渲染清晰
* 开启抗锯齿
* pixelRatio 控制在合理范围，避免过重

例如：

* `antialias: true`
* `setPixelRatio(Math.min(window.devicePixelRatio, 2))`

### controls 要求

* 支持阻尼
* 缩放可用
* 平移可用
* 不要自动旋转（本次基础版先不做）

## 十、必须处理的异常情况

请处理以下异常情况：

### 1. 模型加载失败

若某个 glb 文件加载失败：

* 不要让页面 JS 崩掉
* 在查看器区域显示简单文本提示，例如：

  * “模型加载失败，请检查文件路径”
* 控制台打印清晰错误信息，方便后续排查

### 2. 容器尺寸变化

若弹层打开后页面尺寸变化，查看器需要响应 `resize`：

* 更新 camera.aspect
* 更新 projectionMatrix
* 更新 renderer size

### 3. 重复打开不同模型

如果用户先打开太和殿，再关闭，再打开角楼：

* 必须正常切换
* 不能残留上一个模型
* 不能出现多个 canvas 叠加
